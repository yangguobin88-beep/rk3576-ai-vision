## 一、CPU / NPU / GPU 到底有什么区别？

### 🧠 用一个公司来比喻

想象你开了一家**图片处理公司**：

| 角色 | 对应硬件 | 特点 | 擅长的事 |
|------|----------|------|----------|
| **总经理** | CPU | 什么都能干，但一个人忙不过来 | 协调、决策、处理杂事 |
| **美工团队** | GPU | 人多，干活快，但只会画图 | 图形渲染、游戏画面 |
| **AI 专员** | NPU | 专门做 AI 识别，效率极高 | 人脸识别、物体检测 |

---

### 📦 CPU（中央处理器）

**人话版：**
> CPU 就像公司的**总经理**，什么事都能处理，但一个人忙不过来。

**特点：**
- ✅ 万能，什么都能算
- ✅ 灵活，各种任务都能处理
- ❌ 速度一般，不是专门干 AI 的
- ❌ 跑 AI 模型会很慢、很热

**在 RK3576 上的作用：**
- 启动系统
- 读取摄像头画面
- 把结果画到屏幕上
- 协调 NPU 工作

---

### 🎮 GPU（图形处理器）

**人话版：**
> GPU 就像**100 个美工同时画画**，画得快，但只会画图。

**特点：**
- ✅ 并行能力强（很多人同时干活）
- ✅ 适合训练 AI（在电脑上）
- ❌ 功耗高、发热大
- ❌ 嵌入式设备上很少用 GPU 跑 AI

**你需要知道的：**
- PC 上训练模型用 GPU（NVIDIA 显卡）
- RK3576 上**不用 GPU 跑 AI**，用 NPU

---

### 🤖 NPU（神经网络处理器）

**人话版：**
> NPU 就是**专门干 AI 的机器人**，只会干这一件事，但干得又快又省电。

**特点：**
- ✅ 专为 AI 计算设计
- ✅ 速度快、功耗低
- ✅ 能 7×24 小时跑
- ❌ 只能跑 AI，别的干不了

**在 RK3576 上的作用：**
- **这是你模型跑的地方！**
- 接收 CPU 传来的图片
- 快速识别出"这是人/车/缺陷"
- 把结果返回给 CPU

---

### 🔄 它们怎么配合工作？

```
摄像头 → [CPU 读取图片] → [NPU 识别] → [CPU 画框显示] → 屏幕
           ↑                    ↑              ↑
         协调员              AI专员         协调员
```

**流程说明：**
1. **CPU** 从摄像头读取一帧画面
2. **CPU** 把画面送给 **NPU**
3. **NPU** 快速识别，返回结果（比如"检测到人脸，坐标是 xxx"）
4. **CPU** 根据结果在画面上画框
5. **CPU** 把画面显示到屏幕上

## 二、模型 / 推理 / 训练 是什么关系？

### 🍳 用"厨师"来比喻

| 概念 | 比喻 | 说明 |
|------|------|------|
| **模型** | 菜谱 | 一套写好的"怎么识别"的规则 |
| **训练** | 写菜谱 | 让 AI 学习，生成菜谱的过程 |
| **推理** | 按菜谱做菜 | 拿着菜谱，快速做出结果 |

---

### 📚 模型（Model）

**人话版：**
> 模型就是一本**"怎么识别东西"的说明书**。

**你会接触到的模型文件：**
- `.pt` 文件 → PyTorch 格式（PC 上用）
- `.onnx` 文件 → 中间格式（用于转换）
- `.rknn` 文件 → RK3576 专用格式

**比如 YOLOv8n 模型：**
- 大小约 6MB
- 能识别 80 种物体（人、车、狗、杯子...）
- 这本"说明书"已经写好了，你直接用

---

### 🏋️ 训练（Training）

**人话版：**
> 训练就是**教 AI 学习**的过程。

**怎么训练？**
1. 你给 AI 看 1000 张人脸照片
2. 告诉它"这些是人脸"
3. AI 自己总结规律
4. 最后生成一个模型文件（.pt）

**训练在哪里做？**
- ✅ 在你的 **电脑** 上
- ✅ 有显卡会更快
- ❌ **不在 RK3576 上训练**（板子算力不够）

**你什么时候需要训练？**
- 官方模型识别不了你要的东西时
- 比如你要识别"特定的零件缺陷"

---

### 🚀 推理（Inference）

**人话版：**
> 推理就是**用训练好的模型来识别东西**。

**推理在哪里做？**
- PC 上可以推理（你测试的时候）
- **RK3576 上主要做推理**（真正部署的时候）

**推理的特点：**
- 速度快（毫秒级）
- 只需要模型文件
- 不需要原来的训练数据

**推理流程：**
```
输入图片 → 模型（.rknn）→ 输出结果
                ↓
        "检测到：人脸，置信度 95%"
```

---

### 🔄 三者的关系图

```
┌─────────────────────────────────────────────────────────┐
│                      【你的电脑】                        │
│                                                         │
│   训练数据          训练过程           模型文件          │
│   (1000张图片) ───→ (教AI学习) ───→ (yolov8n.pt)       │
│                         ↑                               │
│                     GPU 加速                            │
└─────────────────────────────────────────────────────────┘
                          │
                          │ 转换（RKNN-Toolkit2）
                          ↓
┌─────────────────────────────────────────────────────────┐
│                     【RK3576 开发板】                    │
│                                                         │
│   摄像头图片        推理过程           识别结果          │
│   (实时画面) ───→ (NPU运算) ───→ (人脸，置信度95%)     │
│                        ↑                                │
│                   模型文件                              │
│               (yolov8n.rknn)                           │
└─────────────────────────────────────────────────────────┘
```

## 三、RK3576 在整个系统里扮演什么角色？

### 🎯 一句话总结

> **RK3576 是一个"能跑 AI 的小电脑"，负责在现场实时识别东西。**

---

### 📋 RK3576 基本参数（看个大概就行）

| 项目 | 参数 | 人话 |
|------|------|------|
| CPU | 8核 ARM | 相当于一台中端手机 |
| NPU | 6 TOPS | 能跑中等大小的 AI 模型 |
| 内存 | 通常 4-8GB | 够用 |
| 功耗 | 约 5-10W | 很省电，可以一直开着 |

---

### 🏭 RK3576 在系统里的位置

```
┌─────────────────────────────────────────────────────────────┐
│                       【完整系统】                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   摄像头 ──┬──→ RK3576 开发板 ──┬──→ 显示屏               │
│            │         ↓          │                          │
│            │    [NPU 识别]      ├──→ 云平台/服务器         │
│            │         ↓          │                          │
│            │    检测结果        └──→ 报警器/继电器         │
│            │                                               │
└─────────────────────────────────────────────────────────────┘
```

**RK3576 负责的事：**
1. ✅ 接收摄像头画面
2. ✅ 用 NPU 跑 AI 模型
3. ✅ 输出识别结果
4. ✅ 可选：显示画面、上传云端、触发报警

**RK3576 不负责的事：**
- ❌ 训练模型（在你电脑上做）
- ❌ 复杂的业务逻辑（通常由服务器做）

---

### 📊 RK3576 能力边界（重要！）

| 场景 | RK3576 能否胜任 | 说明 |
|------|----------------|------|
| 单路摄像头 + YOLOv8n | ✅ 轻松 | 30FPS 没问题 |
| 双路摄像头 + YOLOv8n | ✅ 可以 | 需要优化 |
| 四路摄像头 | ⚠️ 勉强 | 可能掉帧 |
| 大模型（>100MB） | ❌ 不行 | 需要更强的板子 |
| 训练模型 | ❌ 不行 | 在电脑上训练 |

---

## 四、单路 / 多路视频是什么意思？

### 📹 人话解释

| 术语 | 意思 | 例子 |
|------|------|------|
| 单路 | 1 个摄像头 | 门口装一个摄像头 |
| 双路 | 2 个摄像头 | 前门+后门各一个 |
| 多路 | 3个及以上 | 工厂车间多角度监控 |

### ⚠️ 为什么要关心这个？

- 路数越多，NPU 压力越大
- RK3576 适合 1～2 路
- 4 路以上建议用更强的板子（如 RK3588）
